# v0.1.0-alpha.6x-wave3

Wave 3 remediation release for low-priority findings from the `v0.1.0-alpha.6x` audit map.

## Scope

Implemented items:
- `L-01`, `L-03`, `L-04`, `L-06`, `L-07`, `L-08`, `L-09`, `L-10`, `L-11`, `L-12`, `L-13`, `L-14`, `L-15`, `L-16`, `L-17`, `L-18`, `L-19`
- `L-02` intentionally removed as stale/no-op after verification.

## What Changed

### C1 Identity/Crypto hardening
- `KeyPair::from_seed([u8; 32])` is now fallible and rejects all-zero seed (`CryptoError::AllZeroSeed`).
- `NodeIdentity` now enforces invariant safety by keeping `id` and `key_pair` private with read-only getters.
- `PeerId::try_from_public_key_bytes()` added and wired into protocol entry points.
- `PeerId::try_from_public_key_bytes()` now also validates Ed25519 public-key parsing.
- `MessageEnvelope::sender_peer_id()` now returns `Result<PeerId, CryptoError>` so it works in all profiles (including `--no-default-features`).
- Removed deprecated compatibility constructor `MessageEnvelope::new(PeerId, ...)` to eliminate sender-identity ambiguity.

### C2 Compression/precision guards
- Added `CompressionError::InvalidScale`.
- `Quantized` scale validity (`is_finite && > 0`) is now enforced in both `compress()` and `decompress()` paths.
- `sqrt_f32` precision contract documented and covered by convergence test.

### C3 Observe bounds + algorithm scaffolding
- Added fail-closed human-readable serialization guard for ID widths above 16 bytes.
- Added record-bound validators for span/event/metric names and attribute maps.
- Added `MetricRecord.unit` length validation (`MAX_METRIC_UNIT_LEN`).
- Wired record validation into artifact emitter write path (`RunArtifactSink` and direct append methods).
- Marked PSO scaffolding (`ParticleSwarmConfig`, `Particle`) as deprecated (no removal yet).
- Added explicit hot-path note that `Display/Debug` formatting remains stack-buffer based.

### C4 Membership/mock realism
- `PeerId` now derives `PartialOrd/Ord`.
- `MembershipView` switched from `Vec` to `BTreeSet` for active/suspected peer lookup.
- `MockNetwork` now derives deterministic high-entropy peer IDs via `PeerId::from_public_key`.
- Added bounded queue depth (`max_queue_depth`, default 1024) and `MockTransportError::QueueFull`.
- `MockNetwork::send()` is now fallible (`Result<(), MockTransportError>`).

### C5 Model/API semver batch
- `SwarmModel::forward` changed to:
  - `fn forward(&self, input: &Self::Input) -> Result<Self::Output, Self::Error>`
  - Added associated type `Error`.
- `SwarmOptimizer` and `FitnessFunction` now use associated `Model` and associated `Error` types.
- `LinearModel` moved to contiguous parameter storage (`params: [f32; 144]`) so bias is included in parameter roundtrip.
- `LinearModel` internal fields are now private, enforcing construction through validated constructors.

### C6 Cleanup/docs
- Deprecated panic-path `ReplayProtection::with_config` in favor of `try_with_config`.
- Case-safe rename: `readme.md` -> `README.md`; updated roadmap/context links.
- Documented `u32` timestamp overflow horizon (2106-02-07) in crypto API docs and ADR notes.

## Breaking Changes

Batched into this semver-labeled release set:
- `KeyPair::from_seed` now returns `Result`.
- `NodeIdentity` field visibility tightened (`id`, `key_pair` private).
- `MessageEnvelope::sender_peer_id()` now returns `Result`.
- `MockNetwork::send()` now returns `Result`.
- `SwarmModel`, `SwarmOptimizer`, `FitnessFunction` trait signatures changed.
- `LinearModel` public parameter layout changed from split arrays to flat buffer.
- Deprecated `MessageEnvelope::new(PeerId, ...)` compatibility path removed.
