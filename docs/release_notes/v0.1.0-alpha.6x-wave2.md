# v0.1.0-alpha.6x — Wave 2 Remediation

> Additive hardening pass. New enum variants (`ReplayConfigError::ToleranceWindowTooLarge`,
> `CompressionError::UnsupportedMethod`) may require downstream `match` updates.
> All existing function signatures remain unchanged.

## Core Safety (Commit 1)

### M-03: Replay tolerance configurability
- Renamed `SEQUENCE_TOLERANCE_WINDOW` → `DEFAULT_SEQUENCE_TOLERANCE_WINDOW` (backward-compat re-export).
- Added `tolerance_window` field to `ReplayProtection`.
- New `try_with_tolerance_window(capacity, clock_skew, window)` constructor with `MAX_SEQUENCE_TOLERANCE_WINDOW = 256` bound.
- New `ReplayConfigError::ToleranceWindowTooLarge` variant.
- 3 regression tests: configurable smaller window, zero-tolerance strict monotonic, max-exceeded rejection.

### NEW-04: FedAvg numerical precision
- Changed aggregation loop from per-element `+= gradient / n` to sum-then-divide.
- New `fedavg_sum_then_divide_precision` test validates ≤ 1e-6 relative error vs f64 oracle.

### NEW-06: Compression fail-closed
- Replaced wildcard fallback (`_ => compress(…, None)`) with explicit `RandomSparse` and `TopKQuantized` arms returning `CompressionError::UnsupportedMethod`.
- 2 new tests confirm `UnsupportedMethod` behavior.

## DataOps Hardening (Commit 2)

### M-09: NodeV1 write-path validation
- New `validate_node_v1()` function with bounds constants: `MAX_NODE_KEY_LEN = 256`, `MAX_OP_TYPE_LEN = 128`, `MAX_NODE_INPUTS = 256`, `MAX_NODE_OUTPUTS = 256`, `MAX_PARAMS_COUNT = 128`.
- `NodeValidationError` enum with 5 variants.
- Write-path only — `normalize_node_v1()` and read path remain tolerant.
- 4 tests.

### M-12: Shared descriptor bounds + report read-path wiring
- Extracted `check_descriptor_bounds()` shared core from `sanitize_source_descriptor_v0()`.
- New `validate_source_descriptor_bounds()` public API.
- `load_report()` now calls `validate_source_descriptor_bounds()` on all `registry.datasets[].source` entries with warn-and-continue policy.
- 2 new bounds validation tests.

## Validated Constructors (Commit 3)

### M-13: SwarmConfig try_build
- New `SwarmConfigError` enum (`MaxRoundsZero`, `ConvergenceThresholdNotFinite`, `ConvergenceThresholdNegative`).
- New `SwarmConfig::validate()` and `SwarmConfigBuilder::try_build()`.
- Uses `std::result::Result` to avoid crate-level `Result<T>` alias collision.
- 4 tests.

### M-14: Topology try_gossip / try_hierarchical
- New `TopologyError` enum (`FanoutZero`, `FanoutTooLarge`, `LayersZero`, `LayersTooLarge`).
- `MAX_GOSSIP_FANOUT = 64`, `MAX_HIERARCHY_LAYERS = 16`.
- 6 tests.

## TrimmedMean Docs (Commit 4)

### M-10: TrimmedMean rounding docs + edge-case tests
- Added doc comments explaining 0.49 clamping, truncation rounding, and `InsufficientUpdates` guard reachability.
- 2 edge-case tests: clamp behavior, direct-construction guard triggering.

## Build Hygiene (Commit 5)

### NEW-08: async-trait feature gating
- `async-trait` dependency made optional, activated by `std` feature in `swarm-torch-net`.
- Already properly `#[cfg(feature = "std")]`-gated at usage site.

### M-08: Node-ID collision doc
- Added collision probability analysis to `node_id_from_key()` doc comment (128-bit birthday bound).

## Post-Review Patches (Wave 2.1)

### M-09 Enforcement (write-path)
- `RunArtifactBundle::write_graph()` now calls `validate_node_v1()` before normalization/persist (fail-closed).
- Oversized `NodeV1` fields (node_key, op_type, inputs, outputs, params) are now rejected with `io::Error(InvalidData)`.
- 1 new test: `write_graph_rejects_oversized_node_key`.

### NEW-06 Decompress Fail-Closed
- `decompress()` sparse arm now uses explicit method matching instead of wildcard `_`.
- `RandomSparse` and `TopKQuantized` methods with sparse data return `CompressionError::UnsupportedMethod` instead of silently decoding.
- 2 new tests: `decompress_sparse_random_sparse_returns_unsupported`, `decompress_sparse_topk_quantized_returns_unsupported`.
