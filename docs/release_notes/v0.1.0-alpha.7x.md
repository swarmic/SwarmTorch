# v0.1.0-alpha.7x

Wave 7 delivery release: reliability hardening, sequential graph execution, typed ADR-0017 deferred fields, and robustness harness coverage.

## Scope

Implemented:
- Reliability hardening on artifact write/read paths (all-or-nothing materialization staging, snapshot pair mismatch fallback, stricter manifest checks, surfaced load warnings).
- Sequential scheduler MVP with deterministic topological ordering, cycle detection, fail-closed missing-input handling, and execution policy enforcement before runner invocation.
- Typed `NodeV1` deferred fields in `run_graph` with computed `op_hash`.
- Byzantine robustness harness in `swarm-torch-core/tests` with deterministic attack scenarios and aggregator comparisons.

Out of scope:
- Distributed orchestration/scheduler.
- Non-sequential execution backends.
- Any breaking migration of historical bundle formats.

## What Changed

### 1) Reliability Hardening (Wave7 Phase 0)

- Added VS Code rust-analyzer workspace defaults in `.vscode/settings.json` (`cargo.allFeatures`, `check.allTargets`) to reduce local analyzer drift against cargo truth.
- Hardened `DataOpsSession::materialize_node_outputs(...)` in `swarm-torch/src/artifacts/session.rs` with prepare-then-commit staging so multi-output materialization is all-or-nothing per invocation.
- Added snapshot pair consistency marker handling and read-path replay fallback warnings in `swarm-torch/src/report/load.rs`.
- Strengthened manifest validation in `swarm-torch/src/artifacts/bundle.rs` (duplicate path rejection and required-entry presence checks).
- Added `load_report_with_warnings(...) -> io::Result<(Report, Vec<LoadWarning>)>` while keeping `load_report(...)` as compatibility wrapper.
- Optimized node unsafe derivation in `swarm-torch/src/report/model.rs` via registry trust pre-indexing.

### 2) Sequential Execution Engine MVP (Wave7 Phase 1)

- Added new `swarm-torch/src/scheduler.rs` and re-export via `swarm-torch/src/lib.rs`.
- Implemented deterministic topo-sort (`node_key` tie-break), cycle detection, and edge reference validation.
- Implemented `execute_graph_sequential(...)` orchestrator:
  - resolves node inputs from session registry,
  - enforces `ExecutionPolicy` before runner invocation,
  - executes with `NativeOpRunner`,
  - materializes outputs through `DataOpsSession`,
  - emits deterministic scheduler telemetry events,
  - records error/skipped materialization status for fail-closed behavior.

### 3) Typed Deferred Node Fields (Wave7 Phase 2)

- Extended `swarm-torch-core/src/run_graph.rs` `NodeV1` with typed optional fields:
  - `cache_policy: Option<CachePolicyV0>`
  - `materialization_policy: Option<MaterializationPolicyV0>`
  - `resources: Option<ResourceHintsV0>`
  - `op_hash: Option<String>` (computed from canonical op definition)
- Added serde-default/backward-compatible behavior for existing serialized graphs.
- Updated prelude exports in `swarm-torch-core/src/lib.rs` and adjusted call-sites/tests.

### 4) Robustness Harness (Wave7 Phase 3)

- Added `swarm-torch-core/tests/robustness_harness.rs`.
- Added deterministic attack families:
  - outlier gradient attack,
  - sign-flip attack,
  - NaN/Inf injection,
  - collusion cluster attack.
- Added comparative assertions across `FedAvg`, `TrimmedMean`, `Median`, and `Krum`.

## API and Schema Notes

- `DataOpsSession::materialize_node_outputs(...)` now accepts `impl Into<CacheDecisionV0>`; legacy bool behavior remains available via compatibility wrapper `materialize_node_outputs_cache_hit(...)`.
- `report::load_report_with_warnings(...)` surfaces non-fatal load diagnostics (`LoadWarning`) while preserving legacy `load_report(...)`.
- `NodeV1` schema now includes typed optional execution/resource policy fields and computed `op_hash`.
