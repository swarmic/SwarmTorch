# v0.1.0-alpha.6x-wave4

Wave 4 remediation convergence release for `E-01..E-05` from the `v0.1.0-alpha.6x` remediation map.

## Scope

Implemented items:
- `E-01` TopK selection-path throughput improvement with deterministic tie handling.
- `E-02` stronger deterministic seeded init (`SplitMix64`) for `LinearModel::with_random_init`.
- `E-03` const-generic `LinearModel` capacity evolution with default-preserving ergonomics.
- `E-04` richer error context/source chaining across crypto/compression/protocol paths.
- `E-05` `OpRunner::Error` contract tightening via `OpRunnerError` trait bounds.

## What Changed

### C1 — TopK compression throughput (`E-01`)
- Replaced full-sort TopK selection path with `select_nth_unstable_by` partition selection.
- Added deterministic tie ordering by `(abs(value) desc, index asc)`.
- Stabilized wire output by sorting selected sparse entries by index before serialization.

### C2 — Stronger seeded model init (`E-02`)
- Replaced LCG seeded initialization in `LinearModel::with_random_init` with local `SplitMix64`.
- Preserved deterministic reproducibility for identical seeds.

### C3 — Model representation evolution (`E-03`)
- Added const-generic capacity model form:
  - `LinearModelWithCapacity<const MAX_PARAMS: usize = 144, const MAX_OUTPUT: usize = 16>`
- Kept backward-compatible default ergonomics via alias:
  - `type LinearModel = LinearModelWithCapacity<144, 16>`
  - `LinearModel::new(8, 16)` remains valid.
- Preserved contiguous parameter semantics (weights + bias in one backing array).

### C4 — OpRunner error contract (`E-05`)
- Added `OpRunnerError` marker trait in execution boundary:
  - always: `Debug + Display + Send + Sync + 'static`
  - with `std`: also `std::error::Error`
- Tightened `OpRunner` contract to `type Error: OpRunnerError`.

### C5 — Error context/chaining (`E-04`)
- Added `Display` + `std::error::Error` for `CryptoError`.
- Added `std::error::Error` for `CompressionError`.
- Improved `VerifyError` display formatting to use lower-level `Display`.
- Extended `VerifyError::source()` to chain `Crypto` errors.
- Added `std::error::Error` for `SwarmConfigError`.

### C6 — Convergence truth-sync
- Added Wave 4 release-note index entry.
- Updated roadmap/docs to state that Wave 4 sign-off opens the v0.2 F-track.
- Added v0.2 roadmap bullets for `F1` and `F2a/F2/F3/F4` rollout alignment.
