# v0.1.0-alpha.6x-wave5

Wave 5 post-remediation feature rollout for the v0.2 F-track (`F1`, `F2a`, `F2`, `F3`, `F4`).

## Scope

Implemented items:
- `F1` run-graph execution hints (`ExecutionHint`) with canonical hash exclusion.
- `F2a` transform-audit metadata plumbing into artifact materialization and report load/render paths.
- `F2` alloc-gated `UpdateTransform` trait operating on `GradientUpdate`.
- `F3` composable `AggregationPipeline` with shared compression/error paths.
- `F4` lightweight integer-only aggregation tracing estimates.
- Phase-0 doc convergence updates for remediation/backlog plan status.

## What Changed

### F1 — Execution hints in `NodeV1`
- Added `PreferredProfile`, `DeviceAffinity`, and `ExecutionHint` in `run_graph.rs`.
- Extended `NodeV1` with optional `execution_hint` and serde defaults for backward JSON compatibility.
- Kept `execution_hint` excluded from `NodeDefCanonicalV1` and `node_def_hash_v1` cache identity.
- Migrated `NodeV1` call sites in artifacts/report/native runner/example paths.

### F2a — Transform audit plumbing
- Added `TransformAuditV0` to `swarm-torch-core/src/dataops.rs`.
- Extended `MaterializationRecordV2` with `applied_transforms`.
- Added `DataOpsSession::record_transform_applied`.
- Wired `UnsafeReasonV0::UnsafeExtension` propagation from untrusted transform audits.
- Preserved compatibility for legacy records missing `applied_transforms` (`serde(default)`).

### F2 — `UpdateTransform` trait
- Added alloc-gated `UpdateTransform` to `traits.rs`.
- Added `apply_update_transforms` helper in `aggregation.rs`.
- Enforced provenance preservation for `sender`, `sequence`, and `round_id`.

### F3 — Aggregation pipeline
- Added `AggregationPipeline<A: RobustAggregator>` in `aggregation.rs`.
- Added optional compression stage using existing `CompressedGradient` compress/decompress path.
- Added `AggregationPipelineError` and explicit empty-input guard.

### F4 — Lightweight tracing
- Added `AggregationTraceResult`.
- Added `AggregationPipeline::trace(gradient_len, num_peers)` with integer-only estimates:
  - raw memory bytes
  - approximate FLOPs
  - optional compressed-byte estimate for TopK.

### Phase-0 plan/doc convergence
- Updated remediation/backlog plan docs to reflect closure through Wave 4 and post-Wave-4 resume sequencing.
- Updated roadmap/readme/release-note index to reflect this Wave 5 rollout state.
