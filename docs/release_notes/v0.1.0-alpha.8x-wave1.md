## Telemetry Migration Contract (Parser Guidance)

Producer contract remains canonical:

- `swarmtorch.node_key`
- `swarmtorch.detail`
- `swarmtorch.op_type`
- `swarmtorch.input_count`
- `swarmtorch.output_count`

No legacy unprefixed producer keys are emitted by current scheduler/native-runner paths.

Parser migration guidance:

1. Prefer `swarmtorch.*` keys for all new integrations.
2. If historical bundle support is required, treat legacy unprefixed keys as read-only compatibility aliases.
3. Do not emit or persist unprefixed keys in new SwarmTorch-produced artifacts.
