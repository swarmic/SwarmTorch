# Cargo Vet Supply Chain

This directory is the `cargo vet` store for this workspace.

## Files

- `config.toml`: vet policy, imports, and exemptions generated by `cargo vet init`.
- `audits.toml`: first-party audit records.
- `imports.lock`: pinned upstream import metadata.

## Wave 8.1 Policy

- `cargo vet` runs in CI as **advisory** for one wave (8.1).
- `cargo deny` and `cargo audit` are blocking in CI.
- After Wave 8.1, `cargo vet` is intended to be promoted to blocking once the audit baseline is stabilized.
- CI pins supply-chain tool versions:
  - `cargo-deny` `0.19.0`
  - `cargo-audit` `0.22.1`
  - `cargo-vet` `0.10.2`
- First-party audits now exist for critical crates:
  - `ed25519-dalek 2.1.1`
  - `sha2 0.10.9`
  - `postcard 1.1.3`

## Exemption Policy

- Every new `[[exemptions.*]]` entry in `config.toml` must include an explicit
  rationale entry in [`exemption-rationale.md`](exemption-rationale.md).
- Rationale entries must include owner and removal target.
- CI enforces this policy on pull requests.

## Local Commands

```bash
cargo deny check
cargo vet
cargo vet suggest
```
